# 项目组织架构系统

完成项目侦察后、创建或归整员工任务窗口前，使用本协议。组织架构负责定义“谁负责哪类成果”，任务账本负责记录“现在正在做什么、做到哪里”。

## 建立组织架构文件

把 [组织架构模板](../assets/ORG_CHART.template.md) 复制到：

```text
.codex/project-commander/ORG_CHART.md
```

只有总指挥可以写入。默认保持本地且不纳入 Git。未经用户授权，不得修改 `.gitignore` 或提交组织架构文件。不得保存秘密、原始对话全文、隐藏思维过程或复制的文件正文。

出现以下情况时建立或更新：

- 总指挥首次理解项目；
- 项目阶段、主要交付物或风险变化；
- 创建、接纳、改派或释放员工；
- 两个岗位重叠或某项成果无人负责；
- 运行模式改变了合理编制规模。

## 使用两层组织

采用以下层级：

```text
总指挥｜项目名
├─ 治理部门
│  └─ 员工00｜Token监管与模型路由｜项目名
└─ 项目专属交付部门
   ├─ 员工01｜不同职责｜项目名
   ├─ 员工02｜不同职责｜项目名
   └─ 员工NN｜不同职责｜项目名
```

默认不创建“部门经理”任务窗口，由总指挥直接协调所有员工。只有用户明确要求，且额外管理层的协调价值明显高于 Token 成本时，才增加管理层。

部门是组织分组，不能冒充员工。每名真实员工仍必须是 Codex 项目侧边栏中独立、持久化的任务窗口。

## 按交付物设计部门

只建立覆盖不同成果线所需的最小部门集合。根据项目证据调整名称与岗位。

| 项目类型 | 常见交付部门 | 不同成果线 |
| --- | --- | --- |
| 产品/软件 | 产品与规划；架构与开发；质量与发布 | 需求、设计/代码、测试/集成/发布检查 |
| 内容运营 | 调研与策略；内容生产；视觉与分发质检 | 有来源选题、文案/素材、审核/渠道包装 |
| 数据分析 | 数据质量；分析与建模；报告与验证 | 可信输入、计算分析、图表报告与独立检查 |
| 影视媒体 | 前期开发；制作；后期与连续性 | 剧本分镜、拍摄/声音/素材、剪辑/连续性/成片验收 |
| Codex Skill/插件 | 调研与流程设计；技能编写；校验与发布 | 规则来源、SKILL/资源、校验/文档/发布 |
| 业务运营 | 流程与制度；数据与执行；审计与上线 | 批准流程、运营记录/实施、控制/上线 |

混合项目围绕真实交付物建立混合部门，不照搬固定模板。没有已就绪或可预期工作时，不创建空部门。

## 定义每名员工岗位

每名员工只能有一个主岗位。记录：

- 所属部门与员工标题；
- 负责的主要成果；
- 接收什么输入、必须交付什么输出；
- 可写范围与只读依赖；
- 验收责任；
- 基线模型、推理强度和升级条件；
- 一次只能执行一项当前任务；
- 备用或改派规则。

创建或接纳员工前进行岗位独立性检查：

1. 该员工是否拥有其他员工未拥有的成果？
2. 它的可写范围是否与所有并发写入者互不重叠？
3. 总指挥能否明确判断它何时完成？
4. 独立保存上下文的价值是否高于 Token 与协调成本？

任何一项为“否”，就合并或重新设计岗位。不能为了让窗口看起来繁忙而创建两个主成果相同的员工。

节省模式下可以让员工兼任相邻职责，但仍只记录一个主要负责成果；同一工作项的生产者与独立验收者不得由同一员工兼任。

## 任务路由与部门交接

任务账本每一行都必须映射到一个部门和一名唯一负责员工。可以增加只读复核者，但实际生产负责人只能有一名。

所有部门交接由总指挥中转：

1. 来源员工完成并验证自己拥有的产物；
2. 总指挥核验证据并更新任务账本；
3. 总指挥只打包已验收结果、变化事实、文件位置和验收标准；
4. 总指挥把压缩交接包发送给目标员工；
5. 目标员工确认输入并开始一项有边界任务。

员工不得直接改派其他员工或修改组织架构。跨部门决定、冲突和集成全部留在总指挥。

## 按运行模式缩放组织

实际交付员工与必备 Token 监管员工分开计算。

| 模式 | 常见实际交付编制 | 组织策略 |
| --- | --- | --- |
| 节省模式 | 1–2 名 | 谨慎合并相邻职责；高风险时仍保留独立验收 |
| 中等/普通模式 | 2–4 名 | 有价值时拆开规划、生产和验收 |
| 效率模式 | 3–6 名 | 使用更多独立成果线，但必须有已就绪且互不冲突的任务 |

编制规模是由有效工作决定的上限，不是必须填满的配额。持续派工协议中的 WIP 仍控制同时执行的任务数。较大编制可以包含因上下文有复用价值而保留的待命员工。

## 归整长期项目

已有组织架构时：

1. 与当前交付物、项目阶段、文件和任务窗口逐项核对；
2. 保留健康岗位和稳定员工标题；
3. 新任务窗口的历史明确匹配缺失岗位时才纳入；
4. 标记岗位缺口、重叠、过期岗位和用途不明窗口；
5. 先由总指挥发送改派说明，再修改标题；
6. 未经用户明确授权，不归档、替换或接管历史任务；
7. 新派工前同时更新组织架构和任务账本。

除非发生冲突、阻塞或目标变化导致当前组织失效，不要在员工执行任务中途频繁重组。
