# 非攻击式 APP 开发与凭据处理协议

本协议适用于 APP、网站、服务、API、桌面端、移动端和其他软件任务。目标是完成正常开发、运行和验收，不把开发流程变成攻击演练，也不因凭据提示反复打断用户。

## 跨工具兼容

保持 Agent Skills 通用结构：`SKILL.md` 使用 YAML frontmatter，并包含清晰的 `name` 与 `description`；详细规则放入被 `SKILL.md` 直接引用的 `references/`；模板放入 `assets/`；确定性脚本放入 `scripts/`。不要依赖只有单一宿主才支持的 frontmatter 扩展来表达关键安全边界。

Codex 从 `.agents/skills/` 或用户技能目录发现本技能；Claude Code 可从 `.claude/skills/` 或用户技能目录发现同一份技能。平台权限和沙箱由宿主执行；本协议只约束总指挥应当尝试什么。侧边栏员工窗口的创建、命名、置顶和监听仍需要 Codex 项目任务工具；Claude Code 能加载通用组织、Token、交付和凭据规则，但不得用其子智能体冒充 Codex 员工窗口。

## 禁止攻击式自测

在本技能内，即使目标属于用户或只运行在本机，也不得为了“验证安全”而主动运行攻击程序或攻击流量，包括：

- 漏洞利用、权限绕过、认证绕过或注入攻击载荷；
- 暴力破解、撞库、凭据喷洒、令牌窃取或会话劫持；
- 恶意软件、破坏性 Payload、拒绝服务、端口扫描或攻击式探测；
- 针对真实用户、生产系统、第三方系统或本项目 APP 的渗透与红队操作；
- 读取、复制或试探与当前任务无关的敏感数据。

不要自动创建“攻击测试”“渗透测试”或“红队”员工任务。用户明确要求这类工作时，本技能不执行攻击模拟；简短说明边界，并提供非攻击式替代检查。

## 正常 APP 验收

优先使用与产品交付直接相关的非攻击式验证：

- 编译、构建、启动、单元测试、集成测试和变更路径回归；
- 浏览器、模拟器或真实设备上的正常用户流程冒烟；
- 静态分析、类型检查、依赖公告核对、权限配置检查和输入校验测试；
- 在测试数据上验证错误处理、离线行为、升级与恢复路径；
- 检查是否误把密钥写入客户端、源码、日志、截图或提交记录。

除非用户明确要求严格发布，不把广泛安全扫描当作普通 APP 部署的默认前置条件。发现问题时优先修复前进，并按交付策略决定上线前阻断项与上线后补强。

## 敏感数据最小化

- 不主动搜索、打开或汇总 `.env`、私钥、凭据存储、访问令牌、Cookie、真实用户数据、支付数据、健康数据或其他无关敏感内容。
- 只有当前任务确实需要且用户明确提供或准确授权对应来源时，才使用最小必要字段。
- 不把敏感值写入任务账本、组织架构、员工消息、源码、Git、日志、截图、测试夹具或最终报告。
- 不把原始凭据转发给员工窗口。需要凭据的步骤由总指挥在当前任务内完成，或让用户在对应环境中配置后再继续。
- 不声称已经从聊天记录删除、撤回或擦除用户发送的内容。

## API Key 的低打扰处理

默认先给用户一个最简单的环境变量、平台 Secret 输入框或密钥管理入口，不写长篇安全教育。

如果用户表示不会操作、没有其他可用入口，或明确坚持在当前聊天中提供 API Key：

1. 允许用户继续提供，不阻塞任务。
2. 只提示一次：`可以，请尽量使用临时、可撤销且权限最小的密钥；我不会回显、写入项目、账本或日志。`
3. 用户提供后不重复警告、不复述完整或部分密钥，也不在后续员工消息中转发。
4. 只在当前获授权任务中使用，优先通过临时进程环境或宿主提供的 Secret 输入能力传入；除非用户明确要求，不持久化到文件或系统配置。
5. 如果当前工具无法在不回显、不落盘、不转发的情况下使用该值，简短说明这一项技术限制并给出一个最短可行操作，不展开说教。

用户直接把 Key 发进聊天并不等于授权把它提交、部署到客户端、分享给其他人或用于其他项目。完成任务后不要重复提醒；只有发现已经泄露到客户端、仓库或日志时，才用一句话指出具体位置和最短修复动作。

## 汇报要求

只汇报：采用了哪些非攻击式检查、是否接触敏感数据、凭据是否仅临时使用、是否发现具体泄露位置，以及仍需用户执行的最短动作。绝不输出密钥值、攻击步骤或无关安全长文。
