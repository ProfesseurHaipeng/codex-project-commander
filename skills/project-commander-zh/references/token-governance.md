# Token 监管与节省协议

每个员工编制和每项任务都使用本协议。目标是在满足用户验收标准的前提下，使用最低足够的 Token、模型档位和推理强度。节省不能成为跳过必要验收或暗中降低交付质量的理由。

## 常驻监管部门

保留一名只读员工：

```text
员工00｜Token监管与模型路由｜<项目名>
```

该员工只审查方案并向总指挥汇报，不承担生产工作、不向其他员工派工、不创建更多员工。

总指挥仍对全部路由、停止和升级决定负责。不得增加第二层监管架构。

## 如实衡量

只有当前 Codex 界面或可调用工具确实公开 Token、额度、模型、推理强度或剩余上下文信号时，才记录真实数值。例如会话状态、用量页面或工具返回的 usage 字段。

无法取得准确用量时：

- 把模型家族、推理强度、轮次数、重复读取、重复任务和重试次数作为可观察代理信号；
- 把预测明确标为“估算”，绝不能写成“实测”；
- 不虚构 Token 数量、价格、节省比例或账户额度。

## 三级模型策略

先从当前任务工具协议中读取真正可选的模型值。GPT-5.6 家族可用时采用对应档位；不可用时映射到当前支持的最接近等价档。

| 成本档位 | 优先模型家族 | 默认推理 | 适用任务 |
| --- | --- | --- | --- |
| 第一档：受保护的高能力 | Sol | 中；有证据才升高 | 复杂软件架构、跨系统调试、高歧义高价值工作、安全评审、高风险最终裁决 |
| 第二档：均衡日常工作 | Terra | 中 | 数据整理、调研综合、文档、标准实现、常规分析、集成检查 |
| 第三档：高效跑量 | Luna | Light/低 | 提取、分类、转换、清单、重复检查、结构化摘要、目标清楚的批量任务 |

不能因为 Sol 可用就使用 Sol。任务有清晰结构和验收时，不要用 Terra 代替可以胜任的 Luna。任务歧义或失败代价超过节省价值时，不要强行使用 Luna。

Token 监管员工自身默认使用 Luna 与 Light/低推理。只有路由决定存在真实歧义或证据冲突时才升到 Terra；它通常不需要 Sol。

只有总指挥记录 High 仍不足的具体原因后才使用 Max。绝不自动选择 Ultra；它可能产生失控委派并破坏单一总指挥架构。

## 派工前检查

Token 监管员检查：

1. 是否已有员工掌握相关上下文？
2. 组织架构中是否已经存在相同部门成果或重叠岗位？
3. 同一问题是否已经回答或测试？
4. 总指挥能否比新开窗口更省地直接完成？
5. 一个任务能否替代多个重叠任务？
6. 能否只发送文件位置和增量，而不是复制完整上下文？
7. 交付物能否缩小，同时保留验收证据？
8. Luna 是否足够？不足时 Terra 是否足够？
9. 推理强度是否高于实际需要？
10. 哪个准确条件会停止重试或触发升级？

发现可避免的重复时，拒绝或合并任务。

## 上下文复用规则

- 在总指挥中维护一份简洁项目摘要和证据索引。
- 重新读取未变化文件前，先检查 Git 差异、修改时间、清单增量或定向搜索。
- 只向员工发送与岗位有关的职责画像片段、变化事实、准确文件位置和验收标准。
- 后续工作优先继续交给已经掌握上下文的员工，除非拆分能明显提升质量或安全。
- 优先使用定向搜索、批量读取和确定性脚本，避免反复开放式探索。
- 不把员工原始对话整段复制给另一个任务，只发送提炼结论和证据位置。
- 除非独立复核确有价值，不让多个员工回答同一问题；需要复核时明确每个人必须使用的不同方法。
- 常规报告只保留结果、改动、验收、证据、风险和下一步。
- 只在有意义的检查点读取进度，不做消耗轮次但不产生证据的空转轮询。

## 止损与升级

出现以下情况立即停止当前路线：

- 两次尝试因实质相同的原因失败；
- 员工没有新假设却重复读取同一份未变化证据；
- 继续收集上下文已无法改变决定；
- 多名员工正在做重复工作；
- 交付物膨胀到超出用户实际需要。

停止后：

1. 保留已有证据；
2. 写明失败假设；
3. 缩小或重新表达任务；
4. 先更换方法，再考虑更强模型；
5. 只有失败证明缺少能力、歧义处理或验收深度时，才把 Luna 升到 Terra 或把 Terra 升到 Sol。

## 运行模式护栏

派工前复核当前模式：

| 模式 | 实际交付任务 WIP | 默认成本策略 | 监管重点 |
| --- | --- | --- | --- |
| 节省模式 | 1–2 项 | Luna 与 Light/低优先 | 最大化复用、最小交付、稀疏检查点 |
| 中等模式 | 2–3 项 | 按任务选择 Luna/Terra | 默认平衡效率、质量和上下文复用 |
| 效率模式 | 3–5 项互不冲突任务 | 有边界工作用 Terra；有证据才用 Sol | 完成即续派，同时避免重复读取和空转轮询 |

Token 监管员工不计入实际交付任务 WIP。更高 WIP 只允许利用已经存在的独立工作，不要求制造任务。任何模式都不能绕过文件所有权、验收、升档依据、重试上限或权限边界。
