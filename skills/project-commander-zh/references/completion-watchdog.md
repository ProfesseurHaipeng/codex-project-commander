# 完成监听与自动续派协议

独立 Codex 任务窗口不会把完成结果主动写入总指挥任务。桌面通知只提醒用户，不等于总指挥收到员工报告。总指挥必须主动监听员工任务状态。

## 首选：总指挥任务心跳

首次派出生产任务前，检查当前工具是否提供任务心跳或自动化更新能力。可用时：

1. 创建或复用且只保留一个附着于当前总指挥任务的心跳，命名为 `总指挥监听｜<项目名>`。
2. 使用线程心跳，不使用每次新建独立任务的定时任务。心跳必须回到当前总指挥任务并复用其上下文。
3. 在任务账本记录监听器名称、状态、最后巡检时间和下次巡检策略。不要把内部自动化 ID 当作秘密，但也不要无意义地向员工广播。
4. 按运行模式选择最低足够频率：节省模式约 8–10 分钟，中等/普通模式约 3–5 分钟，效率模式约 1–2 分钟。工具只支持其他合法间隔时，选择最接近且不造成空转的间隔。
5. 使用自动化工具的结构化参数，不手写或向用户展示原始 RRULE。
6. 不创建第二个监听器；模式变化时更新现有监听器。

心跳提示必须要求总指挥：

```text
这是总指挥完成监听轮次。只检查当前项目组织架构和任务账本中的员工任务与已就绪队列。

1. 列出同一准确项目路径下的相关任务窗口。
2. 只读取账本所列员工的最新状态和最后一份员工报告。
3. 与上次已处理的员工报告标识、任务 ID 和证据比较；不得重复验收同一报告。
4. 发现完成或待复核时，验证证据，更新任务账本，释放拥有范围，重新计算依赖，并立即续派一项已就绪且适配的任务。
5. 发现受阻时，仅在现有权限内解决；需要用户决定时停止续派并报告最小问题。
6. 对每个看似空闲的员工执行空闲容量审计：先排除未处理报告、漏派任务、受阻、待验收或仍在运行；存在岗位适配且已就绪的任务时立即派发，否则标记休息及原因。
7. 没有实质变化时不向员工发消息、不重复总结、不制造工作。
8. 没有工作中、待验收或已就绪任务时暂停本监听器；仍需用户决定时保留账本并暂停。
```

## 完成检测与去重

把以下条件同时满足视为可处理完成事件：

- 员工任务不再运行，或最新轮次明确结束；
- 最新响应含标准 `员工报告`；
- 报告任务 ID 与账本当前任务一致；
- 报告标识或最后响应时间晚于账本中的“最后已处理报告”。

在任务账本记录每名员工最后已处理的任务 ID、报告时间或稳定响应标识。一个报告只能验收一次。员工仅说“完成”而没有证据时，改为待验收并索取缺失证据，不能直接续派覆盖原任务。

## 验收与立即续派

检测到新报告后，在同一次总指挥轮次内完成：

1. 读取完整员工报告与必要证据；
2. 在真实验收表面核对结果；
3. 更新任务状态和最后已处理报告；
4. 释放写入所有权并重新计算依赖；
5. 从待派队列选择价值最高且岗位适配的任务；
6. 用 `send_message_to_thread` 向空闲员工发送下一份任务合同；
7. 记录派工结果，再决定是否需要向用户汇报。

不要等待其他无关员工完成。不要要求员工自己寻找下一项任务，也不要让员工互相派工。

## 空闲容量审计

每次心跳处理完新报告后，检查花名册中看似空闲的员工。必须先读取任务账本和该窗口最新状态，再分类：

| 判定 | 可观察证据 | 总指挥动作 |
| --- | --- | --- |
| 未处理完成 | 窗口轮次已结束，但最新员工报告尚未验收 | 先按完成协议验收，不能覆盖原任务 |
| 派工漏发 | 账本已分配当前任务，但没有成功的任务合同消息 | 立即补发一次并记录派工结果 |
| 仍在工作 | 窗口正在运行，或尚未到承诺检查点 | 保持工作中，不重复派工 |
| 待验收 | 有报告但缺证据或验收未结束 | 完成验收或索取缺失证据，不派新主任务 |
| 受阻/等待依赖 | 有明确阻塞、依赖、权限或外部状态 | 记录受阻；只解决权限内问题 |
| 可补派 | 无当前任务，且存在岗位适配、依赖已完成、治理门获准、写入范围不冲突并且 WIP 允许的待派任务 | 在本轮立即发送任务合同并改为工作中 |
| 真正空闲 | 无当前任务，也没有满足上述条件的适配待派任务 | 标记 `休息` 和原因，不发送无意义待命消息 |

按以下顺序选择补派任务：已在账本分配但漏发的任务、能解锁关键依赖的任务、与员工现有上下文最接近的任务、用户价值更高的任务、最低足够模型即可完成的任务。

“有工作”只指已经就绪、权限具备、岗位适配、没有写入冲突且不突破当前模式 WIP 的真实任务。不得为了填满窗口而拆出无价值任务、提前派发被依赖或治理门阻塞的工作，或让 Token 监管员承担生产。真正空闲不是停滞或偷懒；休息窗口不计入 WIP，也不需要周期性唤醒消息。新任务进入待派或依赖解除时，下一次心跳重新审计并尽快补派。

## 无心跳能力时的降级

当前界面没有心跳/自动化工具、工作区禁用 Scheduled，或创建失败时：

1. 不得声称已经实现跨任务自动汇报。
2. 在当前总指挥轮次仍活跃时，用项目任务列表和 `read_thread` 进行低频监听并执行相同的空闲容量审计；每次检查间隔保持在约 30–60 秒，并在没有变化时退避，避免 Token 空转。
3. 只读取账本中非终态员工，不扫描无关历史任务。
4. 当前轮次结束前，向用户准确说明：员工结果仍保留在各自窗口，跨轮次自动接力无法保证。
5. 在账本把监听状态记为 `需要人工唤醒`。用户发送“继续巡检”时，先补收所有未处理报告并恢复持续派工，不创建第二名总指挥。

## 停止与恢复

出现以下任一条件时暂停唯一监听器：

- 所有必需任务完成或经授权取消；
- 没有工作中、待验收或已就绪任务；
- 下一步只等待用户选择、权限、凭据或外部状态；
- 用户明确要求停止监听。

恢复总指挥时先读取账本，再扫描所有非终态员工任务，补收停用期间完成但尚未处理的报告，然后重新启用同一个监听器。未经用户明确要求，不删除其他自动化，也不创建独立 Scheduled 收件箱任务代替总指挥心跳。
