# Project Commander 多平台使用指南

## 先判断你用的是宿主还是模型

Codex、Claude Code、OpenCode、Kimi Code、Hermes 是可以运行工具和加载 SKILL 的 Agent 宿主。OpenAI、MiniMax、DeepSeek、豆包通常是模型或 API 提供商。模型负责思考，宿主负责读文件、运行工具、保存会话和创建任务；两者不能混为一谈。

因此本 SKILL 有三种运行方式：

1. 完整员工窗口：宿主具备持久命名项目任务、读写消息、重命名和监听工具。
2. 持久会话总指挥：宿主能保存会话但没有 Codex 等价侧边栏员工。
3. 单会话总指挥：仅通过 SKILL 或便携提示词维护项目画像、组织、账本和队列。

## Codex 与 OpenAI

把中文版安装到 `~/.agents/skills/project-commander-zh`。在 Codex 桌面端项目任务工具齐全时，可使用完整员工窗口模式；Codex CLI 或其他缺少窗口工具的界面会自动降级。

OpenAI API 本身不会加载本地 SKILL 或创建任务窗口。请让能够调用 OpenAI API 的 Agent 宿主加载本 SKILL；若宿主只接受 system/developer instructions，使用 `skills/project-commander-zh/assets/PORTABLE_COMMANDER_PROMPT.md`。

## Claude Code（Cloud Code）

把技能复制到 `~/.claude/skills/project-commander-zh`。技能会原生加载组织、Token、账本、安全和交付规则；若没有 Codex 等价项目任务工具，则采用持久会话或单会话模式，不把 Claude 子智能体冒充员工窗口。

## OpenCode（OpenCloud）

OpenCode 可发现 `.opencode/skills`、`.agents/skills` 与 `.claude/skills`。推荐安装到 `~/.config/opencode/skills/project-commander-zh`，或直接复用 `~/.agents/skills`。根据 OpenCode 当前真实会话工具选择 B/C 级模式。

## Kimi Code 与 Kimi 模型

把技能复制到 `~/.kimi-code/skills/project-commander-zh` 或 `~/.agents/skills/project-commander-zh`，并可输入 `/skill:project-commander-zh` 显式调用。Kimi API 单独使用时属于模型提供商，必须由兼容宿主或便携提示词承载。

## MiniMax 与 MiniMax Codex

MiniMax 可以通过 OpenAI/Anthropic 兼容接口作为多种 Agent 工具的模型提供商。MiniMax 官方也提供 Codex 自定义 provider 配置说明：先按官方页面把 MiniMax 接入 Codex，再把本 SKILL 安装到 Codex 技能目录。此组合就是本仓库所称的“MiniMax Codex”。员工窗口能力仍由 Codex 当前界面决定，不由 MiniMax 模型决定。

不要在 SKILL 中硬编码 MiniMax 当前模型名；模型目录变化时，Sol/Terra/Luna 按实际可选模型重新映射。

## DeepSeek（DeepSea）与豆包

DeepSeek 可由兼容 OpenAI/Anthropic 的 Agent 宿主承载；豆包/火山方舟可由 OpenAI SDK 兼容宿主承载。两者作为纯 API 时都不会自行读取 `SKILL.md`。安装技能到宿主，再在宿主里配置提供商。

## Hermes

Hermes 原生支持 GitHub SKILL 安装：

```bash
hermes skills install ProfesseurHaipeng/codex-project-commander/skills/project-commander-zh
```

Hermes 原生会话可使用持久会话总指挥模式。通过 Hermes OpenAI 兼容 API 接入仅提供聊天的前端时，默认采用单会话模式；只有前端或控制面真实暴露项目任务能力时才升级。

## 跨提供商 Token 路由

- Luna：完成清晰重复工作所需的最低成本选项。
- Terra：有边界日常分析、数据、文档与常规实现的平衡选项。
- Sol：复杂软件、跨系统工作或有依据的高风险裁决。

如果宿主只提供一个模型，就通过推理强度、WIP、上下文压缩和验收深度控制消耗，不虚构模型切换。两次实质相同失败后停止原路线并重规划。
